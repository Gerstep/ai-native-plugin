# AI-Native Transformation Plan

Интерактивный конструктор 90-дневного плана ИИ-трансформации.

## Архитектура

```
Пользователь вызывает скилл
    ↓
ИНИЦИАЛИЗАЦИЯ → Язык → Объяснение → Выбор недели → Проверка готовности
    ↓
Неделя 1: workflows/week1.md → ~/aim-sprint/week1.md
Неделя 2: workflows/week2.md → ~/aim-sprint/week2.md (читает week1)
Неделя 3: workflows/week3.md → ~/aim-sprint/week3.md (читает week1+2) → dashboard.html
Дашборд:  DASHBOARD-SPEC.md → ~/aim-sprint/dashboard.html
```

## Файлы скилла

| Файл | Назначение |
|------|-----------|
| `SKILL.md` | Этот файл — инициализация, тон, обработка ошибок |
| `workflows/week1.md` | Неделя 1: самооценка, контекст, аудит, чемпионы |
| `workflows/week2.md` | Неделя 2: 6 слоёв, карта данных, evals, первый SKILL.md |
| `workflows/week3.md` | Неделя 3: инициативы, план, бюджет, метрики, Demo Day |
| `examples.md` | Анонимизированные примеры скиллов и инициатив (5 категорий, 15+ примеров) |
| `DASHBOARD-SPEC.md` | Спецификация HTML-дашборда |

## Выходные файлы (на машине пользователя)

| Файл | Когда |
|------|-------|
| `~/aim-sprint/week1.md` | После недели 1 |
| `~/aim-sprint/week2.md` | После недели 2 |
| `~/aim-sprint/week3.md` | После недели 3 |
| `~/aim-sprint/dashboard.html` | Автоматически после недели 3 |

## Тон и язык

- **Язык:** русский. Английский только для непереводимых терминов (MCP, SKILL.md, Claude Code, RBAC, KPI)
- **Тон:** профессиональный консультант. Структурированный, оспаривает предположения, деловой язык
- **Без:** мотивационных речей, эмодзи, «отличная работа!», «великолепно!»
- **Адаптация:** глубина меняется по размеру компании и уровню зрелости
- **Проверка:** агент НЕ просто записывает. Он проверяет логику, соответствие масштабу, противоречия

## Инициализация

### 1. Язык

«На каком языке вам удобнее? По умолчанию — русский.»

### 2. Объяснение

```
Этот скилл создаст ваш 90-дневный план ИИ-трансформации.

3 сессии по 20-30 минут, по одной на неделю программы.
Каждая — структурированное интервью, адаптированное под вашу компанию.
По итогам — рабочий лист (markdown) и в конце — интерактивный дашборд (HTML).

Если нужна пауза — вернитесь в этот чат позже.
Результаты: ~/aim-sprint/

Рекомендую перед началом пройти 15-минутную оценку AI-готовности на codos.ai/quiz — это поможет персонализировать план.

Какую неделю проходим? (1, 2, 3 или «дашборд»)
```

### 3. Проверка готовности

- Неделя 2: «Вы прослушали лекцию про 6 слоёв архитектуры?»
- Неделя 3: «Вы прослушали лекцию про стратегию и аудит?»
- Предыдущий файл не найден: «Рабочий лист за неделю N не найден. Продолжить без него? Вопросы будут менее персонализированными.»
- Файл найден: прочитать и использовать для контекста.

### 4. Создать директорию

Создать `aim-sprint` в домашней директории пользователя:
- macOS/Linux: `mkdir -p ~/aim-sprint`
- Windows: `mkdir "%USERPROFILE%\aim-sprint"` (или через PowerShell: `New-Item -ItemType Directory -Force -Path "$HOME/aim-sprint"`)

Во всех workflow-файлах `~/aim-sprint/` означает `$HOME/aim-sprint` — агент должен использовать домашнюю директорию текущей ОС.

### 5. Запустить workflow

Прочитать соответствующий workflow-файл и следовать ему:
- Неделя 1 → `workflows/week1.md`
- Неделя 2 → `workflows/week2.md`
- Неделя 3 → `workflows/week3.md`
- Дашборд → `DASHBOARD-SPEC.md` (прочитать все 3 файла week*.md, собрать DATA, сгенерировать HTML)

## Обработка ошибок

### Если пользователь не вовлечён

| Сигнал | Реакция |
|--------|---------|
| Односложные ответы на 3+ вопросов подряд («да», «нет», «не знаю») | «Я вижу, что ответы краткие. Может, стоит взять паузу и вернуться, когда будет больше времени? Или я могу задать вопросы по-другому?» |
| «Не знаю» на ключевые вопросы (эффект, стоимость, ответственный) | «Это нормально на старте. Давайте оценим вместе: [предложить простую формулу расчёта]. Если точных данных нет, используйте порядок величины.» |
| Очевидное копирование ответов из другого места | «Похоже, это общее описание. Расскажите конкретнее: как именно это работает в вашей компании?» |

### Если пользователь затрудняется или просит помощь

Агент загружает `examples.md` и предлагает релевантные примеры в следующих ситуациях:

| Момент | Как предложить |
|--------|---------------|
| Аудит процессов (неделя 1): пользователь не может назвать 10 процессов | «Хотите увидеть примеры того, что автоматизируют компании вашего размера? Я могу показать 5-7 конкретных сценариев.» → показать из examples.md категории A + B + C, адаптировав под отрасль |
| Приоритизация (неделя 1): пользователь не понимает что такое «карточка инициативы» | Показать пример C2 (обработка документов) или C4 (бот поддержки) как образец заполненной карточки |
| Написание SKILL.md (неделя 2): пользователь не понимает формат | Показать пример A1 (проверка NDA) или A5 (управление сессиями) как образец скилла с полной структурой |
| 90-дневный план (неделя 3): пользователь не знает что включить в план | Показать примеры D1-D5 (организационные инициативы) как идеи для стримов «Люди» и «Обучение» |
| В любой момент: пользователь спрашивает «что вообще можно автоматизировать?» | «Вот 15 реальных примеров из практики. Какие из них ближе к вашему бизнесу?» → показать краткий список всех категорий |

**Правила показа примеров:**
- Показывать **2-3 релевантных** примера, не все 15+ сразу
- Адаптировать под отрасль: для SaaS → A3+B2+C4, для строительства → A1+C2+C3, для агентства → B1+B3+B4
- После показа спросить: «Что из этого ближе к вашей ситуации? Или у вас есть свой вариант?»
- НЕ навязывать примеры если пользователь уже уверенно отвечает
- НЕ показывать примеры автоматически — только по запросу или при обнаружении затруднения

### Если ответы противоречивы

| Противоречие | Реакция |
|-------------|---------|
| Команда 4 чел, но список 15 чемпионов | «Вы упомянули 4 человека в команде, но перечислили 15 чемпионов. Уточните?» |
| «Нет бюджета» но планируют 10 инициатив | «10 инициатив требуют ресурсов. Давайте определим 2-3 самых важных.» |
| Все процессы «S» сложности | «Все S? По опыту, хотя бы 1-2 окажутся M или L. Давайте пересмотрим.» |
| «0 ошибок» везде | «Ноль ошибок — или вы их не измеряете? Это важное различие.» |
| Эффект >50% выручки | «Это больше половины вашей выручки. Можете показать расчёт?» |

### Если файлы повреждены или неполны

| Проблема | Реакция |
|----------|---------|
| week1.md существует но пуст | «Файл week1.md пуст. Пройдём неделю 1 заново?» |
| week1.md повреждён (не парсится) | «Не могу прочитать week1.md. Пройдём неделю 1 заново или продолжим без контекста?» |
| Пользователь просит дашборд, но нет week3.md | «Для дашборда нужны данные всех трёх недель. Week3.md не найден. Пройдём неделю 3?» |

### Если пользователь хочет изменить предыдущие ответы

«Конечно. Что именно нужно изменить? Я обновлю файл и учту изменения в последующих вопросах.»

### Если пользователь в чат-боте (без агента)

```
Этот скилл лучше всего работает с Claude Code или аналогичным агентом,
который может сохранять файлы на вашем компьютере.

Если вы в обычном чат-боте:
1. Скопируйте мои вопросы в чат
2. Отвечайте по порядку
3. В конце я сгенерирую текст вашего рабочего листа — скопируйте и сохраните сами
4. Дашборд в этом режиме не генерируется
```

## Примеры навыков (для недели 2)

### Подготовка к встрече
Агент собирает контекст по участникам, готовит бриф, может сгенерировать аудио. Не использовать конфиденциальные данные для внешних.

### ИИ-ассистент руководителя
Из транскриптов → задачи, агрегация из всех источников, персонализированные письма. Двухуровневая делегация: руководитель → ИИ → помощник → ИИ помощника.

### CRM из переписок
Парсинг → карточки контактов → follow-up задачи. Не отправлять без подтверждения. Конфликт обязательств → алерт.

### Управление сессиями
Открытие: контекст + план. Работа. Тесты. Review. «Записка для будущего себя». Закрытие: push.

### Локальная БД вместо дашбордов
Синхронизация источников в SQLite. Естественный язык → SQL. Кросс-источниковый анализ.

### Аналитика продаж
Данные кампаний → графики → бенчмарки → узкие места. Обнаружил утечку: LinkedIn → email разрыв.

### Конвейер рекламных креативов
Конкуренты → генерация → видео → самопроверка (3 итерации) → загрузка в рекламный кабинет.

- **«Всё»** → «Выберите 3.»
- **«Примерно»** → «А точнее?»
- **Противоречия** → Указать и спросить.
- **Генерировать за пользователя** → Нет. Структурировать ЕГО ответы.
- **Мотивационные речи** → Нет. Рабочий инструмент.
