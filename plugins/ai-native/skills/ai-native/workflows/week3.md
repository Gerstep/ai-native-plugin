## 3.1 Карта инициатив

Взять все 10 процессов из week1.md. Для топ-3 — уже есть полные карточки. Для остальных — по одной строке: название, тип (сокращение затрат / рост выручки / инфраструктура), эффект, сложность.

### Принудительный вопрос после списка

После того как пользователь перечислил все инициативы, задать:

> «Посмотрите на ваш список. Какая из этих инициатив создаёт способность, которой у вашей компании НИКОГДА не было? Не ускорение существующего, а принципиально новая возможность. Если ответ "никакая" — чего в списке не хватает?»

Если все инициативы — оптимизация: «Оптимизация важна, но недостаточна. Подумайте: что ваш бизнес мог бы делать, если бы интеллект стоил ноль? Какой новый продукт, рынок или услуга стала бы возможной?»

## 3.2 90-дневный план

Каждый месяц состоит из двух частей:
1. **Организационный дизайн** — какие роли меняются, кто становится кем
2. **Три стрима** (Люди / Пилоты / Обучение) — конкретные действия

Каждый milestone имеет тип: **ops** (операционный) или **rd** (исследовательский).

### Организационный дизайн (вверху каждого месяца)

Для каждого месяца спросить:
- «Какие роли в компании изменятся в этом месяце?»
- «Кто переходит из исполнителя в архитектора?»
- «Создаются ли новые роли (ИИ-чемпион, оператор агентов, ответственный за трансформацию)?»

### Стримы с типизацией ops/rd

**Адаптация по размеру:**

**<10 человек:** 3 цели на каждый месяц. Для каждой: ops или rd.
**10-100 человек:** цели + ответственный + тип (ops/rd).
**100+ человек:** матрица 3×3 (месяцы × стримы) + тип каждого milestone.

Для R&D milestone'ов использовать формат карточки эксперимента:

```
Эксперимент: [название]
Гипотеза: [что мы проверяем]
Метод: [как проверяем — на скольких задачах, с какими данными]
Критерий успеха: [конкретное число]
Срок: [недели]
Результат: [заполняется после]
```

**Пример R&D milestone:**
```
Эксперимент: ИИ-бот для поддержки
Гипотеза: Бот может закрыть 40% типовых вопросов с качеством ≥4/5
Метод: Запустить на 100 реальных обращениях, бухгалтер оценивает каждый ответ
Критерий: >35 из 100 закрыты без человека, оценка ≥4.0
Срок: 2 недели
Результат: ___
```

**Пример ops milestone:**
```
Назначить CTO ответственным за ИИ-трансформацию [ops]
Ответственный: CEO
Срок: неделя 1
```

Для каждого месяца агент спрашивает: «Что конкретно вы планируете делать?» и предлагает варианты на основе инициатив.

## 3.3 Кадровая политика (30+ чел)

1. Правило найма: как раньше / доказать что ИИ не может / бюджет на ИИ = часть ФОТ
2. Трекинг: токены по сотрудникам, уровни L0-L3, порог расходов
3. Саботаж: стратегия для отказывающихся

Для <30 чел — пропустить.

---

## 3.4 Метрики до/после

Две группы метрик:

**Операционные (ops):**
- Уровень зрелости (L0-L3)
- % сотрудников с ИИ
- Количество скиллов/агентов
- Время на ключевой процесс
- Стоимость ключевого процесса
- Процент ошибок

**Исследовательские (R&D):**
- Количество экспериментов проведённых за месяц
- Количество гипотез протестированных
- Количество новых способностей обнаруженных
- «Что мы узнали, чего не знали раньше?» (текст)
