# Неделя 1: Персональная ОС + Аудит процессов

Время: ~20-30 минут. Результат: `~/aim-sprint/week1.md`

---

## 1.0 Оценка AI-готовности (codos.ai/quiz)

Перед началом направить пользователя на стандартизированную оценку:

«Перед тем как мы начнём строить ваш план, пройдите 15-минутную оценку AI-готовности вашей компании на codos.ai/quiz. Это стандартизированная диагностика по 7 направлениям: организация, данные и инфраструктура, разработка, продажи, поддержка, маркетинг, операционка. По итогам вы получите скор, оценку потенциала в долларах и конкретные рекомендации.

После завершения скопируйте ВЕСЬ текст отчёта и вставьте сюда. Я извлеку данные и использую их для персонализации вашего плана.»

**Если пользователь уже прошёл:** «Вставьте текст отчёта.»

**Если не хочет проходить:** «Без проблем. Мы пройдём оценку здесь в разговорном формате.» → перейти к разделу 1.1 (полная версия).

### Парсинг отчёта

Из вставленного текста извлечь:

| Поле | Где в отчёте | Пример |
|------|-------------|--------|
| Общий скор | «AI Readiness Score [число]» | 15 |
| Уровень | «Starting / Emerging / Advancing / Leading» | Starting |
| Годовой потенциал | «Annual AI opportunity $X – $Y» | $865K – $1.9M |
| Скоры по функциям | Заголовки разделов: «Engineering [число]», «Sales [число]» и т.д. | Eng 18, Sales 11, CS 18, Marketing 28, Ops 14 |
| Потенциал по функциям | «$X–$Y Annual AI potential» под каждой функцией | Eng $447K–$970K |
| Топ инициативы | Нумерованные блоки 1/2/3 с приоритетом High/Med/Low | Параллельные сессии (High), AI coding tools (High) |
| Рекомендации Change Management | 4 квадранта: Role Models, Skills, Conviction, Mechanisms | 12 рекомендаций |

**Маппинг скора → уровень скилла:**
- 0-20 → L0 (Наблюдатель)
- 20-40 → L1 (Пользователь)
- 40-60 → L2 (Строитель)
- 60-80 → L3 (Архитектор)
- 80-100 → L3+ (Продвинутый)

После парсинга подтвердить: «Ваш результат: [скор]/100 ([уровень]). Годовой потенциал: [диапазон]. Самое сильное направление: [функция с макс. скором]. Самое слабое: [функция с мин. скором]. Всё верно?»

**Игнорировать при парсинге:** любые CTA, ссылки на бронирование звонков, маркетинговые блоки типа «See the Full Picture» или «Book Your Diagnostic Call».

## 1.1 Контекст компании (сокращённый, если есть quiz)

**Если quiz пройден** — размер команды, функции и базовый уровень уже известны. Спросить только то, чего quiz не покрывает:

1. «В какой отрасли ваша компания и чем конкретно занимается?»
2. «Что является главной болью или ограничением прямо сейчас?»
3. «Какие системы используете (CRM, ERP, управление проектами, бухгалтерия)?»

**Адаптация по размеру** (из quiz или из ответа):
- **50-300 человек:** + «Как устроена организационная структура? Есть ли IT-отдел?»
- **300+ человек:** + «Есть ли существующие ИИ-инициативы? Какие compliance-требования?»

**Если quiz НЕ пройден** — полная версия:

1. «Расскажите, как вы сейчас используете ИИ в работе?»
2. «А ваша команда? Кто использует и для чего?»
3. «Есть ли персональная операционная система, скиллы?»
4. «Как устроены права доступа для ИИ-инструментов?»
5. «Есть ли ИИ-чемпион в компании?»

→ Определить уровень (L0-L3) из ответов.

Далее:
1. «В какой отрасли? Чем занимается?»
2. «Сколько человек и какова ваша роль?»
3. «Главная боль?»

+ адаптация по размеру (как выше).

## 1.2 Аудит процессов

**Если quiz пройден** — использовать данные quiz для предложений:

«По результатам вашей оценки, наибольший потенциал — в направлении [функция с макс. $ потенциалом]. Вот что диагностика рекомендует в первую очередь:»

Показать топ-3 инициативы из quiz для этой функции (только суть, без маркетинговых деталей). Затем спросить:
- «Это актуально для вашей компании?»
- «Что бы вы добавили?»
- «Давайте расширим до 10 процессов по всем функциям.»

Далее предложить процессы по остальным функциям (из quiz: каждая функция имеет топ-3), дополняя по 5 категориям:
- Входящие, Внутренние, Исходящие, Операционные, Продуктовые

**Если quiz НЕ пройден** — агент предлагает 15-20 процессов для отрасли:

Примеры для агента:

Финтех ~100 чел: квалификация лидов, обработка пролонгаций, QA, генерация КП, мониторинг устройств.

Строительство ~50 чел: разбор тендеров, контроль сроков, подготовка актов, закупки, сметы.

Маркетинговое агентство ~30 чел: генерация креативов, анализ кампаний, отчёты клиентам, медиапланирование, конкурентный анализ.

Для каждого процесса: частота, время (ч/нед), кто выполняет, можно ли автоматизировать.

**Проверка:** «50 часов/нед» в команде из 5 → уточнить. «0 ошибок» → «действительно ноль, или не измеряете?»

## 1.3 Приоритизация — топ-3

Из 10 процессов выбрать 3 с наибольшим потенциалом. Для каждого — карточка:

| Поле | Что спросить | Проверка |
|------|-------------|----------|
| Проблема | «Что конкретно плохо?» | Конкретная боль, не «хотелось бы улучшить» |
| Решение | «Что мог бы делать ИИ?» | Если слишком общий — предложить сценарий |
| Эффект | «Сколько сэкономит в год?» | Если >50% бюджета — уточнить. Если есть quiz — сопоставить с $ оценкой из quiz |
| Стоимость | «Люди × месяцы?» | Если «бесплатно» — объяснить скрытые затраты |
| Сложность | S / M / L / XL | Если всё «S» — скорее всего занижает |
| Ответственный | «Кто владеет проектом?» | Конкретный человек |

**Пример карточки:**
```
Процесс: ИИ-голосовой агент для первичных звонков
Проблема: Маленькая команда продаж (8 чел), нужно масштабировать квалификацию лидов.
Решение: ИИ-агент для первичного контакта. Фаза 2: подсказки в реальном времени.
Эффект: ~$480K/год (замена найма 8 сотрудников)
Стоимость: $100-150K (2-3 разработчика × 3 мес)
Сложность: M
Ответственный: Руководитель продаж
Тип: Сокращение затрат
```

## 1.4 ИИ-чемпионы

- **<5 чел:** «ВЫ — спонсор и чемпион. Определим первый пилот.»
- **5-30 чел:** «Кто использует ИИ больше других? Кто понимает процессы?» → 1-2 чемпиона
- **30-100 чел:** «Определим ядро 3-5 чемпионов. Кто из руководителей экспериментирует?»
- **100+ чел:** Спонсор (1) → Ядро (5-7) → Расширенный круг (10-15). Для каждого: имя, роль, почему.

**Если quiz пройден и есть рекомендация ORG_9 (CAIO):** «Ваша оценка рекомендует назначить ответственного за ИИ-трансформацию на уровне CEO, а не CTO. Кто это мог бы быть?»

**Если quiz пройден и есть рекомендация ORG_3 (Champions program):** «Оценка рекомендует программу чемпионов: 2-3 активных пользователя на отдел с еженедельным обменом опытом. У вас есть кандидаты?»

Критерии: строит сам, понимает процессы, имеет полномочия, платит за подписки из кармана.
