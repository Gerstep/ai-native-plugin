# Неделя 2: Архитектура (6 слоёв) + Карта данных

Время: ~20-30 минут. Результат: `~/aim-sprint/week2.md`
Предварительно: прочитать `~/aim-sprint/week1.md` для контекста.

---

## 2.1 Диагностика по 6 слоям

Предполагается, что пользователь прослушал лекцию 2. НЕ объяснять слои.

Для каждого слоя: оценка 1-5, главная проблема, первое действие.

1. Инфраструктура (MCP, модели, gateway, IAM)
2. Данные, память, контекст (онтология, граф знаний)
3. Оркестрация (координация агентов)
4. Навыки, оценка, улучшение (SKILL.md, evals)
5. Люди (роли, чемпионы, обучение)
6. Управление (бюджет, доступ, аудит, compliance)

Определить слабейший слой.

---

## 2.2 Карта данных (из процессов)

Взять топ-3 процессов из week1.md. Для каждого:
1. «Какие данные нужны?» → сущности
2. «Где хранятся?» → системы + тип данных (SQL / стрим / API / файл)
3. «Кто имеет доступ?» → права

Построить: список сущностей (5-10), связи, системы хранения, карту доступа.

**Проверка:** «всё в Excel» для 200 чел → уточнить. «Все имеют доступ ко всему» → предупредить о рисках.

---

## 2.3 Стратегия оценки (Evals)

1. «Как вы будете знать, что агент работает правильно?»

Три уровня:
- **Unit** — один шаг (правильный MCP? корректный ответ?)
- **Integration** — весь процесс (правильный конечный результат?)
- **Business** — бизнес-метрики (ROI, время, ошибки, удовлетворённость)

2. «Для каждого из топ-3 процессов: как измерите успех? Конкретное число.»
3. «Кто ответственный за качество агентов?»

Для 50+ чел: спросить про observability (Arize, Langfuse) или достаточно логов.

**Проверка:** «проверять вручную» для 100+ задач/день → не масштабируется.

---

## 2.4 Архитектурные решения

3 вопроса:

1. **Права доступа:** allow-list или deny-list? → Правильный: allow-list. Если deny-list — объяснить опасность.
2. **Типы агентов:** регулярные (по расписанию), реактивные (по событию), sleep-time (фон), mirror (зеркало сотрудника)?
3. **Агенты на оргчарте:** будут ли видимы как «коллеги» для команды?

---

## 2.5 Первый SKILL.md

Шаблон + 3 примера:

**Проверка NDA:** получить документ → ключевые условия → сравнить с допустимыми → версия с правками. НИКОГДА не подписывать от имени агента. >$500K → юристу.

**Подготовка к встрече:** список участников → контекст из открытых источников → бриф. Не использовать конфиденциальные данные для внешних.

**CRM из переписок:** парсить переписки → карточки контактов → задачи follow-up. Не отправлять без подтверждения. Конфликт обязательств → алерт.

Пользователь заполняет свой SKILL.md для процесса №1 из недели 1.
