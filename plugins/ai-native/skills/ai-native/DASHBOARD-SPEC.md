# Dashboard Specification — Content & Data Structure

> Визуальный стиль: чистый, современный, в духе Stripe Dashboard. Светлый фон (#ffffff), чёткие лейблы, крупная типографика, цветные акценты для статусов и категорий. Без тёмных тем, без градиентных фонов, без «AI-слопа» (generic фиолетовые градиенты, размытые формы, неоновые свечения). Pixel-perfect аккуратность: тонкие разделители, консистентные отступы, одна палитра. Референс: dashboard.stripe.com.

## Источник данных

Дашборд собирается из 3 файлов: `~/aim-sprint/week1.md`, `~/aim-sprint/week2.md`, `~/aim-sprint/week3.md`.
Агент парсит файлы и встраивает данные как `const DATA = {...}` в HTML.

## 4 вкладки

### Вкладка 1: «Обзор»

**Статистические карточки (3 штуки):**
- Команда: [N] человек
- Инициативы: [N]
- ИИ-чемпионы: [N] (спонсор + ядро + расширенный)

**Радар: 6 слоёв** — гексагональная диаграмма, оценки 1-5, подписи на осях

**Контекст компании** — отрасль, роль, главная боль, список сущностей данных

**ИИ-чемпионы** — спонсор (вверху), ядро (ряд), расширенный круг (ряд, если есть)

### Вкладка 2: «Инициативы»

**Фильтры:**
- Все | S | M | L | XL
- Все | 💰 Сокращение затрат | 📈 Рост выручки | 🔧 Инфраструктурный enabler

**Все инициативы раскрываемые** — клик разворачивает карточку:
- Топ-3 (isTop=true): полная карточка с проблема/решение/эффект/стоимость/ответственный
- Остальные: название + тип + эффект + сложность

**Статус:** Идея | Пилот | Продакшен (цветная метка)

**Перевод типов:**
- cost → «Сокращение затрат»
- revenue → «Рост выручки»
- enabler → «Инфраструктура»

### Вкладка 3: «90-дневный план»

**Две секции:**

**Секция A: Операционный план** (Стримы: Люди / Пилоты / Обучение)
- 3 колонки (Месяц 1, 2, 3)
- Каждый milestone — карточка с описанием, ответственным, чекбоксом статуса
- Тип milestone: операционный (ops) или исследовательский (rd) — визуально различаются

**Секция B: Исследовательские цели** (R&D KPI)
- Количество экспериментов / месяц
- Количество итераций на ключевой процесс
- Новые гипотезы протестированные
- Чекбокс: «что нового мы узнали, чего раньше не знали?»

**Чеклист дня 90** — 7-10 пунктов с чекбоксами (сохраняются в localStorage)

### Вкладка 4: «Метрики»

**Таблица метрик:**
- Колонки: Метрика | Сейчас | Цель | Факт (день 90)
- «Факт» — редактируемый (contenteditable, сохраняется в localStorage)
- Разделение: операционные метрики + R&D метрики

**Карточка Demo Day:**
- Название проекта (крупно)
- Проблема → Решение → Ожидаемый результат → Живое демо

## Интерактивность

- Переключение вкладок без перезагрузки
- Фильтрация инициатив
- Раскрытие/сворачивание карточек инициатив
- Чекбоксы в плане — сохраняются в localStorage
- Метрики «Факт» — редактируемые, сохраняются в localStorage
- CSS-анимации при загрузке

## Адаптивность

- Десктоп: полная раскладка
- Планшет (≤1024px): 2 колонки → 1 колонка
- Мобильный (≤768px): вкладки горизонтально скроллируются, карточки стопкой
