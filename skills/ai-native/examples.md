# Примеры скиллов и инициатив для ИИ-трансформации

> Этот файл используется агентом как справочник. Когда пользователь затрудняется с ответом, просит примеры или не понимает, что можно автоматизировать — агент предлагает релевантные примеры из этого файла, адаптируя под отрасль и размер компании пользователя.

---

## Категория A: Персональные скиллы (для руководителя)

### A1. Проверка и подписание NDA
Агент получает договор (PDF/docx), находит ключевые условия (ответственность, IP, неконкуренция, юрисдикция, условия расторжения), сравнивает с вашим списком допустимых условий (хранится как файл контекста), генерирует версию с правками (tracked changes) и чистую PDF с вашей подписью. Вы проверяете, утверждаете, отправляете. Заменяет часы переписки с юристами для стандартных NDA. Важно: агент не должен подписывать документы от своего имени.

### A2. Подготовка к встречам
Перед каждой встречей агент собирает контекст по каждому участнику — профиль, компания, прошлые взаимодействия. Подаёт информацию в несколько моделей для перекрёстного анализа. Генерирует аудио-обзор в формате подкаста — слушаете по дороге на встречу. Можно задавать уточняющие вопросы в интерактивном режиме. Формат подкаста превращает исследование из чтения за столом в пассивное усвоение.

### A3. ИИ начальник штаба
После каждой встречи агент вытаскивает из транскрипта задачи, назначает ответственных, добавляет в единый список приоритетов. Агрегирует задачи из переписок, почты, заметок. Для маркетинга: собирает контакты, обогащает контекстом, пишет персонализированные письма. Планирует встречи, создаёт контент-стратегии. Можно построить двухуровневую делегацию: руководитель → ИИ → помощник-человек → ИИ помощника.

### A4. CRM из переписок
Агент парсит переписки (выбранные чаты и группы), извлекает упоминания людей, компаний, обязательств, запросов. Сопоставляет с транскриптами встреч. Создаёт и обновляет «карточки людей»: имя, компания, роль, о чём говорили, что просили, что вы обещали. Ставит задачи на follow-up. Перед встречей: «что я знаю о человеке X?» Персональная CRM, которая строится сама из естественного общения.

### A5. Управление рабочими сессиями
Открытие сессии: загрузить общий контекст (кто вы, приоритеты, ожидающие решения). Загрузить проект (текущее состояние, решения, блокеры). Показать план с прошлого раза, подтвердить или скорректировать. Выполнить работу. Прогнать тесты/проверки. Анализ сессии: что получилось, что улучшить. Закрытие: обновить документацию, написать «записку для будущего себя» (handoff note). Push в Git. Записка для будущего себя — критический элемент, решает проблему «где я остановился?» при переключении между проектами. Нетехнический руководитель может вести 3-4 параллельных проекта таким образом.

## Категория B: Маркетинг и продажи

### B1. ИИ PR-агент
Определить цель PR и целевой рынок. Агент с доступом к почте исследует контакты журналистов для релевантных изданий. Пишет пресс-материал в нескольких локализованных версиях (адаптирует не только язык, но и угол подачи под рынок). Персонализирует каждый питч под журналиста на основе его недавних статей. Отправляет письма напрямую, управляет ответами. Результат: публикация в СМИ целевого рынка.

### B2. Аналитика продаж и оптимизация кампаний
Агент создаёт локальную среду Python, подтягивает данные исходящих кампаний (email-цепочки, LinkedIn-коннекции, ответы), генерирует код анализа, запускает его, строит графики. Сравнивает результаты с отраслевыми бенчмарками (исследует, что «хорошо» для вашей отрасли). Находит конкретные разрывы — например: контакты, принявшие запрос в LinkedIn, но не ответившие, не получали follow-up email. Утечка выручки, которую люди не замечали.

### B3. Конвейер рекламных креативов
Определить бриф продукта. Агент парсит креативы конкурентов, анализирует стили и сообщения. Генерирует рекламные креативы в стиле вашего бренда. Для видео: раскадровка → сцены → текст → озвучка → музыка → монтаж. Цикл самопроверки: агент проверяет каждый креатив по чеклисту бренда — до 3 итераций автоматически перед показом человеку. Генерирует лендинги. Загружает в рекламный кабинет. Анализирует результаты после запуска.

### B4. Автономный outreach в LinkedIn
Определить критерии целевой аудитории (роль, размер компании, отрасль, география). Агент через браузер ищет подходящие профили, для каждого пишет персонализированное сообщение на основе профиля, недавних постов, контекста компании. Отправляет запрос на подключение. Персонализация — ключевое: каждое сообщение уникальное, не шаблонное. Дополнительно: парсинг входящих сообщений LinkedIn, обогащение контекстом, более релевантные follow-up. Полный pipeline бизнес-развития.

## Категория C: Операционка и данные

### C1. Локальная база данных вместо дашбордов
Определить ключевые источники данных (аналитика, реклама, CRM, платёжная система). Агент пишет скрипты синхронизации в локальную SQLite. Версионирование базы через Git — откат, diff, история бесплатно. Вместо 5 дашбордов спрашиваете агента: «Какой CAC в этом месяце по сравнению с прошлым?» или «Какие ключевые слова упали в ранжировании?» Агент делает SQL-запрос напрямую — компактный вывод, кросс-источниковые JOIN, которые ни один дашборд не может. Аргумент: SQLite лучше MCP-интеграций — мгновенно, без зависимости от сети, diffable, работает офлайн.

### C2. Обработка первичных документов
Агент получает сканы/PDF (накладные, акты, счета-фактуры), извлекает данные (OCR при необходимости), определяет тип документа, извлекает поля (номер, дата, контрагент, суммы), проверяет контрагента в справочнике учётной системы, формирует проводку по шаблону, отправляет на проверку бухгалтеру. Ограничения: не проводить суммы выше порога без проверки, не создавать контрагентов без верификации, при низкой уверенности — эскалация.

### C3. Мониторинг законодательства
Агент мониторит источники правовой информации, выделяет релевантные изменения, генерирует summary + инструкцию для сотрудников, обновляет внутреннюю базу знаний. Предотвращает штрафы от пропущенных изменений + высвобождает десятки часов в неделю у главного специалиста.

### C4. ИИ-бот поддержки клиентов
Фазы внедрения: Фаза 1 — ИИ черновит ответы, человек проверяет перед отправкой (запуск за 2-3 недели). Фаза 2 — ИИ автоматически решает простые тикеты (после месяца данных Фазы 1). Фаза 3 — ИИ обрабатывает 75%+ с эскалацией к человеку. Даже Фаза 1 удваивает-утраивает количество тикетов на сотрудника в день.

## Категория D: Организационные инициативы

### D1. Программа ИИ-чемпионов
Выбрать 2-3 продвинутых пользователя в каждом отделе, дать им подписки на ИИ-инструменты, проводить еженедельные «покажи что построил» сессии. Чемпионы ведут документацию скиллов по функциям. Большинство компаний переходят с менее 25% до 50%+ адоптации за 8 недель.

### D2. Общее ИИ-рабочее пространство
Развернуть единое ИИ-пространство для всей компании. Для технических команд: Obsidian + Claude Code. Для бизнеса: Cowork или аналог. Хранить файлы CLAUDE.md, библиотеки промптов, конфигурации агентов в общем репозитории, чтобы каждый сотрудник начинал с 80% эффективности вместо 0%. Первый шаг: создать общий каталог с описанием 20 ключевых рабочих процессов как ИИ-скиллов.

### D3. ИИ в performance review
Добавить владение ИИ как критерий оценки сотрудников. Измерять output на человека (PR, тикеты, контент), а не часы работы. Команды, которые стимулируют адоптацию ИИ, видят в 3 раза более быстрое внедрение. Один руководитель построил персонального ИИ-начальника штаба — это стало вдохновением для всего C-уровня.

### D4. Подключение систем через API/MCP
Сделать доступными топ-3 наиболее используемых внутренних инструмента через API или MCP. Системы, работающие только через браузер, невидимы для ИИ-агентов — работу навечно делают люди. Один CTO подключил весь стек через MCP за 2 недели и открыл ИИ-доступ ко всем внутренним данным.

### D5. Замер базовых показателей процессов
Замерить время топ-10 повторяющихся рабочих процессов ДО автоматизации. Без базовых показателей невозможно посчитать ROI или приоритизировать. Типичные открытия: продавцы тратят 40% времени на не-продажные задачи, поддержка 30% на поиск информации, инженеры 25% на код-ревью.
